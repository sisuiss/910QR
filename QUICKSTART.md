# クイックスタート：すぐに試す

## � ステップ0：スプレッドシート実装（初回のみ）

### 1. Googleスプレッドシートを作成

1. [Google Sheets](https://sheets.google.com) にアクセス
2. **新しいスプレッドシート** をクリック
3. 名称を「QRクーポン管理」等に変更

### 2. 「Logs」シートを作成

1. デフォルトシート「Sheet1」を **右クリック** → 削除
2. **「+」ボタン** をクリックして新規シートを作成
3. シート名を **「Logs」** に変更

### 3. ヘッダーを設定

`A1:F1` に以下のヘッダーを入力：

| A | B | C | D | E | F |
|---|---|---|---|---|---|
| 発行日時 | ブラウザID | トークン | ステータス | 使用日時 | 備考 |

### 4. Google Apps Script（GAS）を設定

1. スプレッドシートで **メニュー → 拡張機能 → Apps Script** をクリック
2. **Project settings** で スクリプトIDをメモ（後で使用）
3. `Code.gs` という名前のファイルが自動作成されている
4. **[Code.gs](Code.gs) の内容をすべてコピー** (コードブロック全体)
5. GAS エディタに貼り付けて上書き保存

### 5. index.html を GAS にアップロード

1. GAS エディタの左上 **「+」** をクリック → **HTML ファイル**
2. ファイル名を **「index」** に設定
3. **[index.html](index.html) の内容をすべてコピー** して貼り付け
4. 保存

### 6. GAS をデプロイ

1. GAS エディタの右上 **「デプロイ」** ボタンをクリック
2. **「新しいデプロイ」** をクリック
3. 種類：**「Webアプリ」** を選択
4. 実行者：**「自分」** 
5. アクセス権限：**「全員」** （スタッフ・ユーザーが誰でもアクセスするため）
6. **「デプロイ」** をクリック
7. 生成された **Web App URL** をコピー（例：`https://script.google.com/a/macros/...`）

### 7. 動作確認

1. コピーした Web App URL をスマホのブラウザで開く
2. QRコードが表示されることを確認 ✅
3. スプレッドシート「Logs」に行が追加されて、トークンが記録されたことを確認

> **パスワードについて**  
> Code.gs の冒頭で `STAFF_PASSWORD = 'cosme910'` に設定されています  
> 必要に応じて変更してください ⚠️

## 🎯 動作パターン

### パターン1：初回アクセス（キャッシュなし）

```
ページロード
  ↓
キャッシュ確認 → ない
  ↓
GAS側で getUserCouponStatus(BID) を実行
  ↓
status: 'none' → トークン新規発行
  ↓
QRコード表示（ポーリング開始）
```

### パターン2：キャッシュ有り（前回のトークン存在）

```
ページロード
  ↓
キャッシュ確認 → ある！
  ↓
ローカルストレージから即座にトークン復元
  ↓
QRコード表示（数ms以内！）
  ↓
同時にサーバー側で状態確認（非ブロッキング）
  ↓
使用済みなら自動更新
```

### パターン3：使用済み状態

```
スタッフが認証
  ↓
サーバーがステータス更新「未使用」→「使用済み」
  ↓
ユーザーのポーリング（500ms間隔）で検知
  ↓
「✅ 使用済み」画面に自動切り替わり
（ユーザーがボタンをクリック不要！）
```

## ⚡ 速度の最適化

### ページロード時の表示速度

| シーン | 速度 | 理由 |
|---|---|---|
| キャッシュ有り | **1ms以下** | ローカルストレージから即座に復元 |
| キャッシュなし（初回） | **数百ms** | GAS側でトークン生成 + QRコード描画 |
| スタッフ認証後の更新 | **最大500ms** | ポーリング間隔が500ms |

## 🔐 セキュリティ機構

### 1. ブラウザ指紋（Device ID）

```
userAgent + 言語 + タイムゾーン 
  ↓
ハッシュ化 
  ↓
bid_abc123... の形式で BID を生成
  ↓
同じスマホ・ブラウザなら同じBIDが生成される
```

### 2. 1回限りの検証

GAS側で検索時に：
```
SELECT * FROM Logs WHERE ブラウザID = BID AND ステータス != '使用済み'
```

- **未使用クーポン存在** → 「お一人様1回限り」エラー
- **なし** → 新規トークン発行OK

### 3. トークンの一意性

```
tkn_XyZaBc1234Dw  (12文字ランダム)
```

同じトークンが生成される確率は天文学的に低い。

## 💡 トラブルシューティング

### Q: スマホでURL開いても何も表示されない
**A:** 
- インターネット接続を確認
- ブラウザキャッシュをクリア（Ctrl+Shift+Delete または ⌘+Shift+Delete）
- 別のブラウザで試す（Chrome, Safari など別ブラウザ）

### Q: 「お一人様1回限り」が出ない
**A:**
- シークレット/プライベートモードで開く（新しいBIDが生成される）
- 別のブラウザで開く
- テスト用に複数スマホを使うか、複数端末を用意

### Q: スタッフが認証してもユーザー画面が更新されない
**A:**
- ポーリングが実行中か確認（ブラウザコンソール F12）
- 500msの遅延を考慮（約500ms以内に更新される）
- ネットワーク遅延がないか確認
- スプレッドシートでステータスが「使用済み」に更新されているか確認

### Q: QRが表示されない
**A:**
- ブラウザのコンソール（F12）でエラーを確認
- qrcode.js CDN へのアクセスがブロックされていないか確認（企業のファイアウォール等）
- 別のネットワーク（モバイルデータ等）で試す
