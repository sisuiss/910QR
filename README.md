# 店舗クーポン用ワンタイムQRコード認証システム

## 📋 概要

- **要件**: 動的QRコード生成 → スマホカメラでスキャン → スタッフが認証すると次回から使えない
- **構成**: Google Apps Script（GAS）+ Google Sheets + HTML/JavaScript
- **特徴**: 
  - ブラウザ指紋による端末識別
  - ローカルストレージでトークンをキャッシュ（高速表示）
  - リアルタイムポーリング（500ms間隔）で状態同期
  - 2パターンのUIのみ（QR表示画面、使用済み画面）

## 📱 ユーザーフロー（利用者側）

1. WebApp URLにアクセス
2. ページロード時、自動的にトークン発行処理開始
3. QRコードが表示される

### パターンB：使用済み後のアクセス
1. WebApp URLにアクセス

3. 同時にサーバー側で状態確認
4. 使用済みだったため「✅ 使用済み」画面に切り替わる

## 🔐 スタッフの認証フロー

### フロー：QRをスキャン → パスワード認証 → 使用済みに更新

1. **ユーザーがQRをスタッフ端末で読み込む**
   - → スタッフ専用認証ページに自動遷移

2. **スタッフがパスワード「cosme910」を入力**
   - → 「適用」ボタンをクリック

3. **サーバー側で状態更新**
   - スプレッドシートのステータスを「未使用」→「使用済み」に変更
   - 使用日時を記録

4. **ユーザーの画面がリアルタイム更新**
   - ポーリング（500ms間隔）で状態を確認
   - 使用済みを検知 → 約500ms以内に「✅ 使用済み」画面に切り替わる

**列の説明：**
- **A: 発行日時** - トークン発行時刻

- **C: トークン** - QRコードに含まれるランダムトークン
- **D: ステータス** - 「未使用」または「使用済み」
- **E: 使用日時** - スタッフが認証した日時

### Code.gs（バックエンド）

**主要関数：**

| 関数名 | 用途 |
|---|---|
| `doGet(e)` | ウェブアプリのエントリーポイント（パラメータで画面を出し分け） |
| `issueToken(bid)` | クーポンを新規発行、スプレッドシートに記録 |
| `getUserCouponStatus(bid)` | ユーザーの最新クーポン状態を取得 |
| `handleStaffAuth(token, password)` | スタッフ認証、ステータス更新 |
| `checkTokenStatus(token)` | トークンの現在のステータスを返す（ポーリング用） |
| `showStaffAuthScreen(token)` | GAS側でスタッフ認証ページのHTMLを生成 |
| `generateToken()` | ランダムなトークンを生成 |

### index.html（フロントエンド）

**主要機能：**

| 機能 | 説明 |
|---|---|
| ブラウザID生成 | `generateBrowserId()` - userAgent + 言語 + タイムゾーンをハッシュ化 |
| トークンキャッシュ | `localStorage.setItem()` で発行したトークンを保存 |
| ローカルストレージ復元 | DOMContentLoaded時にキャッシュから復元→即座に表示 |
| 画面出し分け | `initializeScreen()` で状態に応じてUIを切り替え |
| QRコード生成 | qrcode.js ライブラリを使用 |
| リアルタイムポーリング | 500ms間隔で `checkTokenStatus()` をコール |
| 自動画面更新 | 使用済み判定で自動的に「✅ 使用済み」に切り替え |

## ⚡ パフォーマンス最適化

### ページロード時の画面表示速度

1. **キャッシュ有の場合**（前回のトークンが存在）
   - HTML読み込み → ローカルストレージから即座に復元（1ms以下）
   - 画面表示 → QRコード生成（数百ms）
   - バックグラウンドで状態確認（非ブロッキング）
   - 使用済みなら自動更新

2. **キャッシュ無の場合**（初回アクセス）
   - HTML読み込み 
   - GASに `getUserCouponStatus()` をコール
   - トークン発行 `issueToken()`
   - QRコード表示

### ポーリング設定

- **間隔**: 500ms（高速応答）
- **トリガー**: トークン発行時に `startPolling()` 開始
- **停止**: 使用済み検知時に `stopPolling()` で自動停止

## 🐛 既知の制限事項

| 制限 | 理由 |
|---|---|
| ブラウザ指紋はシークレットモードで回避可能 | 完全には端末を識別できない |
| スクリーンショット共有で不正利用の可能性 | タイムスタンプのみであり、時間枠での検証なし |
| オフライン環境では動作しない | GAS + Sheets への通信が必須 |
| 複数の同じトークンは並行出力不可 | 設計上：1ユーザー = 1トークンのみ |

## 🔒 セキュリティ考慮事項

1. **ブラウザID検証**: `issueToken()` で未使用クーポンの有無を確認
   ```javascript
   if (data[i][1] === bid && data[i][3] !== '使用済み') {
     return { status: 'error', message: 'お一人様1回限りです。...' };
   }
   ```

2. **パスワード検証**: スタッフ認証でパスワードチェック
   ```javascript
   if (password !== STAFF_PASSWORD) {
     return { status: 'error', message: '❌ パスワードが間違っています' };
   }
   ```

3. **使用済み重複防止**: 既に使用済みのトークンは再度変更不可
   ```javascript
   if (status === '使用済み') {
     return { status: 'error', message: '❌ このクーポンは既に使用されています' };
   }
   ```

## 📊 トラブルシューティング

### Q: スマホで画面が表示されない
**A:** 
- インターネット接続を確認
- ブラウザキャッシュをクリア（Ctrl+Shift+Delete）
- 別のブラウザで試す

### Q: QRが表示されない
**A:**
- ブラウザのコンソール（F12）でエラーをチェック
- qrcode.js CDNへのアクセスが遮断されていないか確認

### Q: 「お一人様1回限りです」が出ない
**A:**
- 別のブラウザ（異なるブラウザID）で試す
- シークレットモードをクリアして再度テスト

### Q: スタッフ認証後、ユーザー画面が更新されない
**A:**
- ポーリング実行中か確認（ブラウザコンソールで `startPolling` の状態をチェック）
- 500msの更新遅延を考慮（最大500ms以内に反応）
- ネットワーク遅延がないか確認

### Q: スプレッドシートに行が追加されない
**A:**
- App Sheets から実行権限が与えられたか確認
- appsscript.json で `"executeAs": "USER_DEPLOYING"` が設定されているか確認
- Code.gs の SHEET 変数が正しくシートを参照しているか確認
